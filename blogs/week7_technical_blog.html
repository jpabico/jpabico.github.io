SQL Injection

April 26, 2014

SQL injection is a technique that exploits security loopholes in data driven applications. Examples of SQL injection can take the form of:

  • Incorrectly filtered escape characters - For example, entering ' or '1'='1 as the userName for the line of code:
        SELECT * FROM users WHERE name ='" + userName + "';
    renders the SQL statement
        SELECT * FROM users WHERE name = '' OR '1'='1';
    If this code was part of an authentication procedure, access would be granted since '1'='1' is always true.
  • Incorrect type handling - For example, entering 1;DROP TABLE users as a_variable for the line of code:
        SELECT * FROM userinfo WHERE id =" + a_variable + ";
    renders the SQL statement
        SELECT * FROM userinfo WHERE id=1;DROP TABLE users;
    This code would result in the server's unintended loss of data.

To help reduce the chances or effects of the negative consequences of SQL injection, some steps that can be taken which include:

  • Use prepared statements - in SQL injection, user input is used as part of the SQL statement. Prepared statements force the user input to be handled as the content of a parameter (and not as part of the SQL command). The query and the data are sent to the SQL server separately using different protocols.
  • Escape user input characters that have special meaning in SQL before putting it in a query
  • Check for patterns - require users to provide input using a specific type and format. Then check to see if the type and input syntax are valid.
