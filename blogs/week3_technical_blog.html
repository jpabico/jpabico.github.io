Looping In Ruby

March 25, 2014

Looping is a technique commonly found in programming languages. It involves repeating a set of commands until the loop is "told" to stop. There are a variety of ways in Ruby to "tell" a loop to stop repeating. Though there are specific parameters and special cases to the following commands/methods, the following are guidelines as to when to use a particular type of loop.

  • "for...in..." loops can be used to repeat a block of code if you know how many repetitions you want ahead of time
  • ".times" is similar to "for..in.." in that you need to specify the number of repetitions ahead of time
  • "while" can be used to perform an unspecified number of loops as long as a condition is true (for example, as long as i > 5 is TRUE, keep repeating)
  • "until" is the opposite of "while". Perform an unspecified number of loops as long as a condition is false (for example, as long as i > 5 is FALSE, keep repeating)
  • ".each" will repeat a block of code for each element in an array and will return the ORIGINAL array at the end
  • ".map" will do what ".each" does and will return a NEW array at the end

It is good practice to check the documentation for the Ruby programming language to understand the specific details of each of the methods above.
